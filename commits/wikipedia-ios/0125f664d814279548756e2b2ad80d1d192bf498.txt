Fix incorrect highlighting of invisible find in page matches.

Should not try to highlight matches in...

- a text node which is not visible because its parent node was hidden
- text which is not visible because it appears in elided text (after the ... ) in elements using 'text-overflow: ellipsis;'

The bug manifested wnen jumping between find in page matches - when trying to jump to an match which wasn't actually visible the article would jump around.

First noticed on `enwiki > Food spoilage` when searching for the term `food`.
- just above the `See also` section there is a hidden div with a text node containing the word `food`
- the description of one of the read more results has the word `food` after the `...` a couple times
