Fix: Capitalize algorithm parameter in QR codes for Google Authenticator compatibility

Google Authenticator fails to parse QR codes when the algorithm parameter
is lowercase (e.g., "sha1"). This fix ensures proper capitalization while
preserving data integrity for exports and metadata updates.

1. QR display: Capitalizes algorithm on-the-fly (sha1 â†’ SHA1)
   - view_qr_page.dart: In-app QR code viewer
   - html_export.dart: HTML export with embedded QR codes
2. User edits: Now use fromAccountAndSecret which generates uppercase
   - setup_enter_secret_key_page.dart: Manual code entry/edit flow
   - code.dart: Added generatedID parameter to fromAccountAndSecret
   - Preserves original type/algorithm/digits/period when editing
   - Only applies isStreamCode logic when creating new codes
3. Metadata updates: copyWith preserves original algorithm case
   - Tags, notes, pinned, trashed, tap count, position, HOTP counter
4. Exports: Preserve original rawData exactly as stored
   - Plain text exports maintain original format
5. New codes: Already using uppercase via fromAccountAndSecret

Code improvements:
- Refactored fromAccountAndSecret to accept optional generatedID parameter
- Correctly preserves original code properties when editing
- isStreamCode logic only applies when creating new codes
- Uses UI values only when creating new codes

This ensures:
- Google Authenticator compatibility for all QR codes
- Historical data integrity (old lowercase codes preserved in storage)
- Exports maintain original format
- User edits preserve all non-editable fields exactly
