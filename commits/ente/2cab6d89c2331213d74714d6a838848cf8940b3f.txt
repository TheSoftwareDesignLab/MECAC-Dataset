Optimize home widget image quality with V2 implementation

Add V2 home widget rendering that uses 1024x1024 images instead of
512x512 thumbnails, enabled in debug mode via kDebugMode flag.

Key Features:
- V2 renders at 1024x1024 using full image files (not thumbnails)
- Uses hw.HomeWidget.renderFlutterWidget() for clean implementation
- Decodes images in isolate via compute() to prevent UI jank
- Automatic fallback to V1 (512px) for non-image files or on error
- Saves 25 images max in V2 vs 50 in V1 (higher quality, fewer images)

Implementation Details:
- Added useWidgetV2 getter to FlagService (returns kDebugMode)
- Created getWidgetImageLimit() method for centralized limit logic
- renderFile() checks flag and uses V2 with automatic V1 fallback
- _captureFileV2() only processes FileType.image files
- Uses ClipRRect widget with rounded corners and BoxFit.cover
- All widget services (memory, album, people) use shared logic

Constants Added:
- THUMBNAIL_SIZE_V2 = 1024.0
- WIDGET_IMAGE_LIMIT_V1 = 50
- WIDGET_IMAGE_LIMIT_V2 = 25
- WIDGET_IMAGE_LIMIT_MINIMAL = 5

Architecture:
- Centralized flag check in renderFile() - no duplication
- Single source of truth for image limits
- Consistent behavior across all widget types
- Self-documenting constants instead of magic numbers
