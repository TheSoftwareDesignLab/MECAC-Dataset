logo: Fix nondeterminism in ImageMagick step

The ImageMagick `convert` command, introduced in b13cf4e78,
apparently inserts the current time into the file.  That makes the
build not reproducible.

Fix that using the "strip-nondeterminism" tool from the
Reproducible Builds project:
  https://reproducible-builds.org/docs/timestamps/

The resulting change to the affected file can be seen using `sng`,
and purely removes a pair of timestamps:

$ f=ios/Runner/Assets.xcassets/AppIcon.appiconset/Icon-1024x1024@1x.png \
  && diff -U3 <(git show @:"$f" | sng) \
              <(<"$f" sng)
--- /dev/fd/63	2025-01-06 21:11:42.422900540 -0800
+++ /dev/fd/62	2025-01-06 21:11:42.426900539 -0800
@@ -13,14 +13,6 @@
