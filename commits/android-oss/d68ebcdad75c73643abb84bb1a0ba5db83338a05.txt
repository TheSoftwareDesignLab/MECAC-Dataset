Moved pledge_minimum label under the Your pledge amount label and added pledge_maximum label that is initially INVISIBLE.
The maximum number of characters a user can enter in the pledge amount is 13. The maximum pledge (numerically) is 1,200,000.00 which is 12 characters including the separators but if the user didn't enter a decimal they could type 1,200,000,000.

Added output pledgeMaximum to PledgeFragmentViewModel that emits the maximum amount a user can pledge (including shipping).
Added output pledgeMaximumIsGone to PledgeFragmentViewModel that emits a Boolean determining if the maximum should be visible to the user. It returns true when the user has entered an amount that would put them over the project's country's currency maximum.
Added output pledgeButtonIsEnabled to PledgeFragmentViewModel that emits a Boolean determining if the Pledge button should be enabled. It returns true when the user has entered a valid pledge amount. It only emits for logged in users pledging or updating their payment method.
Added output continueButtonIsEnabled to PledgeFragmentViewModel that emits a Boolean determining if the Pledge button should be enabled. It returns true when the user is logged out and has entered a valid pledge amount. It only emits for logged out users.
Removed output showMinimumWarning from PledgeFragmentViewModel because we no longer need to warn them if they attempt to pledge an invalid amount; the button will be disabled.
