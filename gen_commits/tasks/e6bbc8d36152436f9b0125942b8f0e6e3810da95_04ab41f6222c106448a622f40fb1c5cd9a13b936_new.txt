fix: Improve widget update reliability and logging

Ensure widgets refresh consistently after data changes or configuration.
- Uses `sendOrderedBroadcast` with a short delay to allow Android to update widget cache before `notifyAppWidgetViewDataChanged`.
- Explicitly calls `notifyAppWidgetViewDataChanged` after `onAppWidgetOptionsChanged`.
- Introduces a cache-buster URI parameter for `RemoteViewsFactory` to force data reloads.
- Adds extensive Timber logging to debug widget lifecycle and update events.
- Refactors widget update calls to use suspend functions and cleans up unused imports.
