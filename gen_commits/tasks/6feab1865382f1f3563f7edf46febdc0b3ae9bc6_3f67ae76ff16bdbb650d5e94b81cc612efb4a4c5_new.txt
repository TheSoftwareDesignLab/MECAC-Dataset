refactor: Centralize task modification date and sync triggers

The `TaskDao.touch` method now automatically updates the task's modification date
and triggers a sync via WorkManager. This change removes scattered manual
`setModificationDate(now())` and `syncAdapters.sync()` calls from various
adapters and the TaskMover.

- `TaskDao.touch` now uses `strftime('%s','now')*1000` for modification dates
  to store milliseconds since epoch.
- `TaskMover` no longer directly handles sync or modification date updates.
- `SyncWork` now explicitly checks if sync is enabled before running.
