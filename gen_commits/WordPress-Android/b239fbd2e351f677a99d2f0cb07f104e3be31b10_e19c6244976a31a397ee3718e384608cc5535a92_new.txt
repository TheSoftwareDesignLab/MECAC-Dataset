refactor: Extract date formatting logic to item mapper

The `BloggingPromptsListItem` Composable previously contained date formatting logic, which coupled the UI to presentation concerns. This change introduces `BloggingPromptsListItemModelMapper` to handle the transformation from `BloggingPromptModel` (domain model) to `BloggingPromptsListItemModel` (UI model).

The mapper now computes and provides the `formattedDate` string directly within the UI model, allowing the Composable to simply display it. This improves the separation of concerns, enhances testability, and aligns better with MVVM architecture.

- Introduce `BloggingPromptsListItemModelMapper` to map domain `BloggingPromptModel` to UI `BloggingPromptsListItemModel`.
- Move date formatting logic from `BloggingPromptsListItem` to `BloggingPromptsListItemModelMapper`.
- Update `BloggingPromptsListItemModel` to include `formattedDate`.
- Modify `BloggingPromptsListViewModel` to use the new mapper when updating UI state.
- Adjust `FetchBloggingPromptsListUseCase` to return raw `BloggingPromptModel`s.
- Add unit tests for `BloggingPromptsListItemModelMapper`.
