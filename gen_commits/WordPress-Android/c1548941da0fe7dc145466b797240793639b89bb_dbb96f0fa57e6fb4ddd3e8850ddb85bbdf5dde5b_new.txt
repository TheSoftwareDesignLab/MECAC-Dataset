refactor(Media): Use standard system window handling in MediaSettingsActivity

Removed the custom `WordPress.NoActionBar.TranslucentStatus` theme and the manual `adjustToolbar()` method.
Enabled `android:fitsSystemWindows="true"` on the activity's root layout and AppBar components.
This simplifies the layout logic by allowing the system to manage status bar insets automatically, reducing custom UI overlap handling.
Also introduces a constant for the image gradient scrim height multiplier.
