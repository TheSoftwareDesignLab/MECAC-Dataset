refactor: Update layout item tap handling to use UiState callbacks

Refactors the layout item tap event handling mechanism.
- Renamed `LayoutListItem` to `LayoutListItemUiState` to better reflect its role.
- Moved the tap event handler into `LayoutListItemUiState` as an `onItemTapped` lambda.
- Removed the `layoutTapped` method from `LayoutSelectionListener` and its implementation in `ModalLayoutPickerFragment`.
- `ModalLayoutPickerViewModel` now directly provides the tap callback when creating `LayoutListItemUiState` objects.
- `LayoutViewHolder` now directly invokes the `onItemTapped` lambda from `LayoutListItemUiState`.

This change simplifies the view layer by embedding event handling directly into the UI state, promoting a more declarative pattern.
