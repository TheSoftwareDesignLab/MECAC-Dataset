feat: Implement wallet error handling; update deps & deployment targets

This commit introduces a new exception handling mechanism for wallets,
allowing external components to register an error callback for wallet-specific
exceptions during operations like `updateAll`.

Key changes include:
- **feat(wallet):** Add `setExceptionHandler` to `WalletBase` and integrate
  it into `ElectrumWalletBase` and `AppStore` to report FlutterErrorDetails.
- **fix(exchange):** Correct `BuildContext` usage in exchange page dialogs
  to ensure `Navigator.pop` operates on the dialog's navigator.
- **chore(build):** Set iOS deployment target to 11.0 and macOS to 12.0.
- **deps:** Bump several CocoaPods dependencies for iOS (CryptoSwift, SDWebImage, SwiftProtobuf)
  and macOS (connectivity_plus_macos, ReachabilitySwift).
- **refactor(responsive):** Update `ResponsiveLayoutUtil` to use
  `MediaQueryData.fromWindow` for screen size detection.
- **clean:** Remove unused imports (`dart:typed_data`, `bitcoin_mnemonic.dart`)
  and the `decrypt.swift` file.
