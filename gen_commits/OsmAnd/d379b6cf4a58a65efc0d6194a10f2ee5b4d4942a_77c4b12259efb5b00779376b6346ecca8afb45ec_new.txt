refactor(ui): Standardize bottom sheet button items and logic

- Removed `bottom_sheet_item_button_with_icon.xml` in favor of a new, unified button layout (`bottom_sheet_button_with_icon`).
- Consolidated button types using `TripRecordingActiveBottomSheet.ItemType` across `ClearRecordedDataBottomSheetFragment` and `StopTrackRecordingBottomFragment`.
- Refactored button click handling from a single `OnClickListener` method to individual listeners per item.
- Moved UI logic for button styling (text color, icon tint, touch states) out of the `ItemType` enum into dedicated methods within `TripRecordingActiveBottomSheet` or configured directly using `UiUtilities`.
- Introduced `DividerSpaceItem` for consistent spacing in bottom sheets.
- Added state saving for the current GPX file in `TripRecordingActiveBottomSheet` to handle configuration changes.
