refactor: Improve context menu item default color handling

Rename `holoLight` parameter to `lightTheme` for better semantic clarity.
Remove explicit `defaultColor` parameter from `ContextMenuItem` setters,
relying on `ContextMenuItem.INVALID_ID` to indicate that the system should
use the theme's default icon color.
Streamline icon color resolution within `ContextMenuArrayAdapter` based on
the current theme.
Eliminate redundant `IconsCache.getDefaultColorRes` calls across various
plugins and map widget configurations.
