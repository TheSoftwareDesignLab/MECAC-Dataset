feat: WikiIndexer: Integrate MySQL for coordinate extraction

Implemented fetching of geographical coordinates for Wiki pages from an external MySQL database (e.g., Toolserver `coord_wikilocale_wiki` tables).
The `WikiIndexer` now includes the `mysql-connector-java` library and connects to a configurable MySQL instance.
Coordinates are no longer parsed heuristically from Wiki text; instead, page IDs are used to query the database, improving accuracy and reliability.
Wiki text parsing is now primarily used for extracting the description if coordinates are found in the database.
Deprecated the old `analyzeTextForGeoInfo` method.

Other changes:
- Updated coordinate input in `NavigatePointActivity` to allow the ':' character for DMS format.
- Reordered POI context menu actions in `POIMapLayer`.
