refactor: Centralize default widget ID processing for availability

The logic to extract the default widget ID from a given ID (stripping any instance-specific suffixes) has been centralized.
- `OsmAndAppCustomization.isWidgetAvailable` now expects the default ID to be passed directly.
- `ApplicationMode.isWidgetAvailable` now explicitly calls `WidgetType.getDefaultWidgetId` when checking default widget availability.
