fix: Improve context menu state retention and map zoom precision

This commit addresses several issues:
- **Context Menu:**
    - Correctly saves and restores the context menu's description across configuration changes.
    - Ensures the `ContextMenuLayer` initializes with previous state (lat/lon, description, selected objects) if available.
- **Map Zoom:**
    - Refines `setZoom` logic to only add a small offset (`0.001f`) when the target zoom is fractional, improving precision.
    - Increases the `isZooming` detection tolerance from `0.0001` to `0.02` for more reliable completion detection.
- **Build Configuration:**
    - Excludes the `rtree/` directory from the `.classpath`.
