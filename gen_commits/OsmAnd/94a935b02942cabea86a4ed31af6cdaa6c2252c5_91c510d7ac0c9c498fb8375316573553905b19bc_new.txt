fix: Truncate coordinate widget text and copy full value

Updates coordinate TextViews in the widget to use `match_parent` for
`layout_width`. This prevents layout issues with long coordinate strings
by truncating the displayed text using a new `AndroidUtils.setTruncatedText`
utility.

The full, untruncated coordinate value is now stored internally and
used when copying coordinates to the clipboard, ensuring complete
information is copied despite UI truncation.
