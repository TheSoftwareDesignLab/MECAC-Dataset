fix: Bottom sheet backgrounds respect night mode in landscape

The `AndroidUtils.setBackground` method was consistently passed `false` for the night mode parameter when setting the background for various map marker bottom sheets in landscape orientation. This prevented the dark theme background from being applied correctly when night mode was active.

This commit replaces the `false` literal with the actual `nightMode` status, ensuring the landscape backgrounds correctly reflect the current theme in:
- `CoordinateInputBottomSheetDialogFragment`
- `HistoryMarkerMenuBottomSheetDialogFragment`
- `OptionsBottomSheetDialogFragment`
- `OrderByBottomSheetDialogFragment`
- `PlanRouteOptionsBottomSheetDialogFragment`
- `SaveAsTrackBottomSheetDialogFragment`
- `ShowDirectionBottomSheetDialogFragment`
