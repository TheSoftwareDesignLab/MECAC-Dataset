fix: Improve UI consistency and night mode support for import screens

Refactors UI elements across import and profile data screens for better
theming and layout consistency.

- Updated fragment backgrounds from `activity_background_basic` to
  `activity_background_color` and toolbar backgrounds from `colorPrimary` to
  `actionModeBackground` in `fragment_import.xml`, `fragment_import_complete.xml`,
  and `fragment_import_duplicates.xml`.
- Refined layout margins for icons/indicators in `list_item_import.xml`,
  `profile_data_list_item_child.xml`, and `profile_data_list_item_group.xml`
  by replacing `layout_margin` with explicit `marginStart`/`End`/`Top`/`Bottom`
  and introducing `list_content_padding_large` for improved spacing.
- Ensured `LayoutInflater` correctly applies night mode themes by using
  `UiUtilities.getInflater(app, nightMode)` in `DuplicatesSettingsAdapter.java`.
