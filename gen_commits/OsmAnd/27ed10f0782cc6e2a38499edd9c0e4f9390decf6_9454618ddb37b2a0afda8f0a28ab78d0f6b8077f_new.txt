refactor: Centralize map view parameters with RotatedTileBox

Introduces a new `RotatedTileBox` class to encapsulate map rendering parameters like tile coordinates, zoom, and rotation. This refactors `ResourceManager` and `RendererLayer` to use this object for consistent map updates and rendering.

Optimizes rotation calculations in `OsmandMapTileView` by pre-calculating sine and cosine values. Improves layer rendering performance by passing pre-calculated `latLonBounds` to `onDraw` methods, avoiding redundant coordinate conversions.

Cleans up redundant local variables in map layers, removes a commented-out TODO, and standardizes the use of integer `getZoom()` for map data calculations. Removes deprecated map rotation key event.
